# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest (v2.9.x) | Yes |
| v2.8.x | Yes |
| v2.7.x | Yes |
| < v2.6.0 | No |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability in FreshKeeper, please report it responsibly.

**DO NOT** open a public GitHub issue for security vulnerabilities.

### How to Report

1. **Email**: Send a report to WuSuBuDuoMing@users.noreply.github.com
2. **GitHub Private Vulnerability Reporting**: Use [GitHub's private vulnerability reporting](https://github.com/WuSuBuDuoMing/freshkeeper/security/advisories/new)

### What to Include

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact assessment
- Suggested fix (if available)

### What to Expect

- **Acknowledgment**: Within 24 hours of your report
- **Triage**: Within 48 hours for critical issues, 7 days for others
- **Resolution**: Critical vulnerabilities addressed within 7 days, non-critical within 30 days
- **Disclosure**: We will coordinate with you on timing of public disclosure

## Security Best Practices

For users of this application:

- Never share your AppID or API keys in public repositories
- Use environment variables or secure storage for sensitive configuration
- Keep your WeChat Developer Tools and base library updated
- Review third-party component permissions before adding them

## Scope

This security policy applies to:
- The FreshKeeper source code in this repository
- Configuration and build files included in the repository

It does not apply to:
- Third-party services or APIs used by the application
- The WeChat platform itself

## Acknowledgments

We appreciate the security research community and will acknowledge contributors who help improve the security of this project (with their permission).

Thank you for helping keep FreshKeeper and its users safe!
